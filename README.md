# Введение

Данный репозиторий представляет собой проект по выполнению индивидуального задания
на обучающем стенде с микроконтроллером LPC2148 с привлечением документации по МК
и методических указаний по выполнению лабораторных работ 2007 года.

Проект делится на два этапа: теоретическую и практическую. В практической необходимо
реализовать на светодиодной матрице 8x8 динамическую индикацию.

# Цель и задачи теоретического этапа

**Цель:**

	- Изучить принципы и механику взаимодействия микроконтроллера с
	периферией.

> Под динамической индикацией понимается смена состояний светодиодов при сигнале
извне во время исполнения программы микроконтроллером.

**Задачи:**

	1. Изучить работу МК с регистрами;
	2. Изучить работу SPI и матрицы;
	3. Изучить работу джойстика;
	4. Изучить работу LCD-дисплея.

# Практическая часть

Поняв, как именно работает МК и вышеперечисленная периферия, как на языке C пишутся программы, можно
приступить к реализации динамической индикации.

Первым делом включается питание стенда. На стенд заливается прошивка через UART и микросхему
преобразования USB-UART с помощью приложения "Flash magic". На LCD-дисплей должно выводиться сообщение
"GNU-Paint|by Golev Andrey", где символ | означает перенос на следующую строку.

МК по SPI отправляет на матрицу данные, какой светодиод будет изначально включён. Пусть это будет
точка в верхнем левом углу (относительно главного вида стенда). Точка будет всё время гореть, пока
на МК не придёт сигнал прерывания со стороны джойстика, который и будет управлять главной точкой.

В зависимости от направления движения, МК должен сразу же перевести точку в нужное место, при этом
оставляя уже зажённый светодиод в текущем состоянии. При нажатии джойстика в центральном положении
подаётся сигнал о том, что ГТ при движении не зажигает светодиоды, при повторном нажатии зажигает.

При нажатии кнопки Button точка должна выключать светодиоды. При нажатии Reset всё сбрасывается, 
точка возвращается в исходное положении.

Также пусть будет гореть один светодиод, который будет оповещать об ожидании ввода данных.

Отрисовка точки будет происходить в обработчике прерывания, где и будет восприниматься сигнал
от джойстика.

# Дополнение

Программа пишется в Keil uVision 3, где формируется Hex-файл для прошивки МК.
Было бы неплохо изучить работу JTAG, чтобы МК не казался чёрным ящиком.

Для этого стоит установить драйвера микроконтроллера на свой компьютер, Keil uVision 3
и Flash Magic для первоначальной проверки работы стенда.
